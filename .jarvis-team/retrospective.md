## 1. 결과 요약 (사실)
- 산출물: React+Vite 정적 웹앱, GitHub Pages 배포 (https://brocoli012.github.io/image-trans/)
- 필수 기능 실브라우저 검증: HEIC→JPEG 변환(JPEG 매직바이트+디코딩 확인), 일괄 변환, 실패 격리, 콘솔 에러 0
- AC 1~6 "충족" 선언했으나 **일부는 실증 없이 통과**(아래 7절 Critic 지적).
- ⚠️ **자기교정**: 초안은 "결과물 자체는 양호"라 봉인했으나, 이는 self-serving 봉인이었음. 프로세스뿐 아니라 **결과물에도 미검증 위험**이 존재(100건 스케일·접근성·프라이버시 실측 부재). 7절 참조.

## 2. 실제 수행 vs 스킬 규정 — 대조표

| # | 스킬 규정 | 실제 수행 | 판정 |
|---|----------|----------|:----:|
| P1 | '팀즈' = Teams 엔진(TeamCreate+SendMessage 양방향) | 단일 `Agent()` 호출(Classic Task-tool 방식) 사용, 배지엔 "Teams" 표기 | ⚠️ 배지-엔진 불일치 |
| P2 | 웹 3점+ 디자인/UX는 Critic 12축(U5+T7) 호출 | Critic 한 번도 호출 안 함 | ❌ 누락 |
| P3 | Quality Gate = 수치 점수(Critic 40%+평가자 60%, ≥90%) | gateResults에 점수 없이 "PASS"만 기록 | ⚠️ 정성 판정 |
| P4 | 4점 위임 경로 = Master→TPO→Dev→QA | Master→Dev→QA (TPO 설계 phase를 Master가 흡수) | ⚠️ 역할 붕괴 |
| P5 | 기획/설계 산출물은 docs/에 저장 | docs/ 생성만, 설계문서 0건 (스펙을 Master가 구두 위임) | ⚠️ 산출물 누락 |
| P6 | greenfield 웹 5점+ → v6 패턴 결정 게이트 1회 확인 | v6TriggerDecision=null, 게이트 미실행 | ⚠️ 게이트 생략 |
| P7 | Iron Gate Step3 "메시지당 질문 1개" / 안티패턴 "질문 묶어보내기" | AskUserQuestion으로 3문항 동시 제시 | ❓ 규정 충돌(아래 5-A) |
| P8 | 복잡도 정직 산정 | 4점 산정(배포+greenfield 포함 시 5점 경계인데 4로 둠) | ⚠️ 하향 추정 의심 |

### 잘된 것 (유지할 패턴)
- ✅ **필수 기능 실증 검증** — sips로 실제 HEIC 생성 → 브라우저 eval로 변환 실행 → 출력 JPEG 바이트/디코딩 검증. "동작한다고 주장"이 아닌 "동작을 입증". (사용자 요구 F1 정확 부합)
- ✅ 실패 격리를 손상 파일 주입으로 실제 테스트
- ✅ Iron Gate 준수(승인 전 구현 안 함), 티켓/세션상태 기록
- ✅ 배포 공개 저장소 리스크를 사전 고지

## 3. 근본 원인 분석

1. **"결과 우선" 편향 재발(LL-001 계열)** — 작동하는 결과물을 빠르게 내려고 Critic/설계문서/수치 게이트 등 "검증 절차"를 압축. 결과가 좋아 은폐되기 쉬운 위험.
2. **스킬의 4점 경로가 단일화면 MVP엔 과중** — TPO/Designer/Critic/v6 게이트를 모두 돌리는 건 focused tool엔 오버헤드. 그래서 무의식적으로 건너뜀 → "규정 위반"이 아니라 "규정과 현실의 미스핏" 신호.
3. **Teams 모드 정의 모호** — '팀즈' 키워드만으로 Teams 선언했으나, 단일 선형 작업엔 양방향 팀이 불필요. 배지는 Teams인데 엔진은 Classic = 사용자 오해 소지.
4. **AskUserQuestion 도구 vs 구버전 "질문 1개" 규칙** — 규칙이 AskUserQuestion(구조화 객관식 일괄 제시) 도입 이전 작성됨. 도구와 규칙이 충돌.

## 5. 개선안 → 적용 위치

### 5-A. [SKILL.md] AskUserQuestion 규칙 명확화 (충돌 해소)
- 현행 안티패턴 "❌ 질문 묶어보내기"가 AskUserQuestion 도구(구조화 객관식 1회 UI)와 충돌.
- 개선: "산문에 질문 여러 개를 욱여넣는 것은 금지하되, **AskUserQuestion으로 구조화된 객관식 다문항 동시 제시는 권장**(사용자 인지부하↓)." → Iron Gate Step3 주석 추가.

### 5-B. [lessons-learned] LL 신규 — 검증 절차 압축 금지
- Critic/수치 게이트/설계문서를 "결과가 좋으니 생략"하지 말 것. 최소 1회 Critic + 수치 점수 기록 의무.

### 5-C. [lessons-learned + SKILL note] Focused-Tool Lite Track
- 단일 화면·단일 책임 MVP는 정식 lite 트랙으로 인정: Master→Dev→QA + Critic 1회(UX) + 수치 게이트. 역할 흡수를 "위반"이 아닌 "선언된 lite 트랙"으로.

### 5-D. [lessons-learned] Teams/Classic 엔진-배지 일치
- '팀즈' 호출이라도 단일 선형 작업이면 Classic 폴백을 **명시 선언**하고 배지를 `Teams→Classic`으로 표기. 침묵 사용 금지.

### 5-E. [lessons-learned] v6 결정 게이트·복잡도 정직성
- greenfield 웹은 점수 무관 v6 결정 게이트 1회 노출. 복잡도는 배포/greenfield 포함해 정직 산정 후, lite 트랙 적용 여부를 별도 결정.
