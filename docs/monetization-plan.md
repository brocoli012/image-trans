## 0. 전제 (이 앱의 구조적 특성)
- **클라이언트 변환** = 서버 연산비 0. 사용자가 100만이어도 변환 비용 안 늘어남.
- 비용은 사실상 **도메인 + 정적 대역폭**뿐. 대역폭도 Cloudflare Pages면 무료.
- 약점: 세션이 짧음(변환 후 바로 이탈) → 방문당 광고 수익이 낮음. **수익의 관건은 "트래픽 확보(SEO)"**.
- 강점이자 긴장: "사진이 기기 밖으로 안 나감"이 USP인데, 광고 추적과 인식상 충돌 → 절제·문맥광고 권장.

## 1. 광고 수익화 전략

### 1-1. 호스팅부터 정리 (선결)
- ⚠️ **GitHub Pages는 상업적/광고 사이트 사용을 ToS로 제한**. 광고 붙이면 위반 소지.
- → **Cloudflare Pages로 이전 권장**: 정적 대역폭 **무제한 무료**, 커스텀 도메인·광고 허용, HTTPS 자동. (대안: Netlify/Vercel = 광고 OK이나 무료 대역폭 100GB/월 상한)

### 1-2. 광고 네트워크 선택
| 네트워크 | 승인 난이도 | RPM 기대 | 비고 |
|---------|:---------:|:-------:|------|
| Google AdSense | 높음 | 중~상 | 표준. 단일 유틸 페이지는 "thin/low-value content"로 **거절 위험** → 콘텐츠 페이지 필수 |
| Ezoic | 낮음 | 중 | 소형 트래픽도 승인 관대, AI 최적화. 초기 현실적 1순위 |
| Media.net | 중 | 중 | 문맥광고, AdSense 보완 |
| Adsterra 등 | 매우 낮음 | 하 | 승인 쉬우나 품질·UX 저하 → 비권장 |

→ **권장: 초기 Ezoic 또는 (콘텐츠 보강 후) AdSense.**

### 1-3. 수익을 실제로 만드는 레버 = SEO 콘텐츠
- 유틸 단일 페이지만으론 ① 광고 승인 어렵고 ② 검색 유입이 없음.
- **가이드/블로그 2~4편 추가**: "HEIC가 뭔가요", "아이폰 사진 JPG로 바꾸는 법", "HEIC vs JPEG 차이" 등.
- 효과: AdSense 승인↑ + 검색 유입↑ + 광고 지면↑. **이게 핵심.**

### 1-4. 광고 배치 (UX 보존)
- 변환 **결과 영역 하단**에 1~2개. 상단 큰 배너·팝업·인터스티셜 금지(부모님 친화 USP 훼손).
- 개인정보처리방침·이용약관 페이지 필수(광고 = 추적 쿠키).

## 2. 서비스명 + 도메인

### 2-1. 네이밍 원칙
짧고 · 기억 쉽고 · 기능/프라이버시 연상 · 글로벌 발음 가능 · 도메인 확보 가능.

### 2-2. 후보 (가용성은 §6에서 확인 필요)

- `HeicEase` · `SnapJPEG` · `JustJPEG` · `PixSwap` · `FormatFox`

**프라이버시 강조형 (USP 부각)**
- `LocalPix` · `OffShot` · `PrivPic` · `OnDevice(.app)`

**한국 타겟 보조**

→ 개인 1순위 제안: **프라이버시형 1개 + 기능형 1개**를 후보로 좁혀 도메인 확인.

### 2-3. 도메인 TLD
| TLD | 연 비용(대략) | 적합성 |
|-----|:-----------:|------|
| .com | $10~15 | 신뢰·SEO 최선, 좋은 이름은 선점됨 |
| **.app** | $14~18 | HTTPS 강제, 툴 이미지에 적합 (추천) |

| .net/.co | $10~30 | .com 대체 |

## 3. 운영 비용 예측 (사용자 규모별)

전제: 앱 번들 ≈ **1.6MB**(heic2any WASM 포함), 신규 방문만 다운로드(캐시 후 ≈0). 변환은 사용자 기기에서 → 서버비 0.

| 월 신규 방문 | 대역폭(≈1.6MB/신규) | Cloudflare Pages | Netlify/Vercel 무료(100GB) | 합계(도메인 포함) |
|---|---|---|---|---|
| 1천 | ~1.6GB | **$0** | $0 | ~$1/월 |
| 1만 | ~16GB | **$0** | $0 | ~$1/월 |
| 10만 | ~160GB | **$0** | 초과(이전/유료) | ~$1/월 |
| 100만 | ~1.6TB | **$0** | 상당한 유료 | ~$1/월 |

- **Cloudflare Pages 사용 시 100만 방문도 호스팅 $0** → 운영비 = 도메인 ~$10~18/년이 사실상 전부.
- 선택적 추가비: 분석(Plausible ~$9/월, 또는 GA 무료), 이메일(무료티어). 광고 네트워크는 무료(수익 배분 모델).
- 비용 폭증 트리거: **서버사이드 기능 추가**(대용량·완전 멀티프레임 변환)를 하면 그때부터 연산/대역폭 비용 발생. 현재 MVP는 해당 없음.

## 4. 예상 수익 (러프 — 반드시 보수적으로)
- 유틸 RPM $1~5(지역 의존, 한국·동남아 낮음/미국 높음), 세션당 노출 1~2.
- 시나리오: 월 10만 방문 × 1.5노출 × $2 RPM/1000 ≈ **$300/월**. 1만 방문 ≈ $30/월. 1천 방문 ≈ $3/월.
- ⚠️ **정직한 경고**: "heic to jpeg" 키워드는 경쟁 포화(거대 경쟁사 다수). **트래픽 확보가 가장 어렵고**, SEO/콘텐츠 없이는 방문 자체가 안 나옴. 광고는 "트래픽이 있을 때" 비로소 의미.

## 5. 추천 로드맵 (단계)
1. **호스팅 이전** GitHub Pages → Cloudflare Pages + 커스텀 도메인
2. **법적 페이지** 개인정보처리방침·이용약관 추가
3. **SEO 콘텐츠** 가이드 2~4편 + 메타태그/사이트맵/OG
4. **광고 승인** Ezoic(초기) → 트래픽 성장 시 AdSense
5. **절제된 광고 배치 + 분석** 측정·최적화
6. (선택) PWA·다국어로 유입 확대
