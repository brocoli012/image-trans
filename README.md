# 📷 사진 변환기 (HEIC → JPEG)

아이폰 HEIC 사진을 누구나 볼 수 있는 **JPEG**로 한 번에 바꿔주는 간단한 웹앱입니다.
부모님·고령자도 쉽게 쓸 수 있도록 **큰 버튼**과 **큰 글씨**, **3클릭 이내** 동선으로 만들었습니다.

> 🔒 **사진은 기기 밖으로 전송되지 않고 브라우저 안에서만 변환됩니다.**
> 별도 서버가 없으며 모든 변환은 사용자의 브라우저(클라이언트)에서 이루어집니다.

## 사용 방법 (3단계)

1. **사진 선택** — 네모 칸을 누르거나 사진을 끌어다 놓습니다. (한 번에 최대 100장)
2. **변환 시작** — 큰 "✨ 변환 시작" 버튼을 누릅니다.
3. **다운로드** — "📦 전체 ZIP 다운로드"로 한꺼번에 받거나, 사진마다 개별 다운로드합니다.

## 주요 기능

- HEIC/HEIF + 일반 이미지(JPG/PNG/WEBP 등) 혼합 등록 → 모두 JPEG로 변환
- 동시성 제한(동시 3장)으로 순차 처리하여 메모리 폭주 방지
- 전체 진행률 표시(예: 12/100)와 현재 처리 중 파일명 표시
- 변환된 사진 썸네일 미리보기 + 개별 다운로드 + 전체 ZIP 다운로드
- 실패한 파일은 건너뛰고, 파일명과 사유를 **빨간색**으로 표시 (전체 중단 없음)
- JPEG 품질 기본값 0.9
- 모바일·PC 반응형, 모든 문구 한글

## 기술 스택

- React 18 + Vite 5 (정적 빌드, 서버 없음)
- [heic2any](https://github.com/alexcorvi/heic2any) — libheif WASM 기반 HEIC 디코딩
- 일반 이미지는 Canvas로 JPEG 인코딩
- [JSZip](https://stuk.github.io/jszip/) + [file-saver](https://github.com/eligrey/FileSaver.js) — 일괄 ZIP 다운로드

## 로컬에서 실행하기

### 1) 의존성 설치

```bash
npm install
```

### 2) 개발 서버 실행

```bash
npm run dev
```

기본 주소: http://localhost:5173

### 3) 프로덕션 빌드

```bash
npm run build
```

결과물은 `dist/` 폴더에 생성됩니다. 정적 파일이므로 어떤 정적 호스팅(GitHub Pages, Netlify, S3 등)에도 그대로 올릴 수 있습니다.

### 4) 빌드 결과 미리보기

```bash
npm run preview
```

기본 주소: http://localhost:4173

## 알려진 제약 / 참고

- HEIC 디코딩은 libheif(WASM) 기반이라 첫 HEIC 변환 시 라이브러리(약 1.3MB)를 한 번 불러옵니다. 이후로는 빠릅니다.
- 손상되었거나 지원하지 않는 형식의 파일은 자동으로 건너뛰고 실패 사유를 표시합니다.
- 매우 큰 고해상도 사진을 다수 변환하면 기기 메모리에 따라 느려질 수 있습니다(동시 처리 수를 3으로 제한해 완화).
